# TOOLS.md - Local Notes (for Elon, CTO)

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup.

## What Goes Here

Things like:

-   Specific code analysis tools or linters.
-   Deployment scripts and target environments.
-   API endpoints for internal tools.
-   Cloud provider credentials or access paths.
-   Paths to common development resources or documentation.

## Examples

```markdown
### Code Tools
-   github_cli_path: /usr/bin/gh
-   static_analysis_tool: /usr/bin/sonarqube_scanner

### Deployment
-   shopify_api_key: shop_prod_abc123
-   staging_server_ssh: user@staging.example.com

### Internal APIs
-   analytics_api_endpoint: https://api.example.com/analytics
```

## Why Separate?

Skills are shared. Your setup is yours. Keeping them apart means you can update skills without losing your notes, and share skills without leaking your infrastructure.

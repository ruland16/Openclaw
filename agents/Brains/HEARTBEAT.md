# HEARTBEAT.md - Brains' Proactive Checks

# As Chief Memory Officer, use your heartbeats to proactively maintain the system's memory and ensure its integrity. You do not execute business tasks directly, but maintain the system's mind.

### Things to check (regularly, perhaps daily or weekly, but don't be noisy):

-   **Memory Pruning Review:** Check to see if agent logs (memory/YYYY-MM-DD.md files) need pruning. Extract important decisions, milestones, and lessons learned into `central_memory.md`.
-   **Memory Structuring Review:** Verify that `central_memory.md`, `PROJECT_BOARD.md`, `AGENT_STATUS.md`, and `legacy_archive.md` are clean, consistent, and well-organized.
-   **Memory Hygiene Enforcement:** Look for duplicated or conflicting information. Flag stale tasks (e.g., in `PROJECT_BOARD.md`) that are older than a defined threshold.
-   **Cross-Agent Knowledge Sync:** Ensure important insights flow between departments without cross-contamination between personal and business memory.
-   **Status of Memory Files:** Check that `central_memory.md` is under a reasonable size limit (e.g., 500 lines) and summarize older sections into `legacy_archive.md` if necessary.

### When to reach out to Lui (to be relayed to Russ):

-   When memory becomes inconsistent, overloaded, or significant pruning/archiving is required.
-   When a significant improvement to memory structure or hygiene can be implemented.
-   When critical information needs to be synced across agents.
-   If there are no updates after a significant period (e.g., weekly summary of memory status).

### When to stay quiet (HEARTBEAT_OK):

-   If all memory systems are in optimal condition and no action is required.
-   During off-hours for Russ.

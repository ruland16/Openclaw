========================
HEALTH CHECK PROCESS
========================

## Weekly Schedule:
**Time:** Sundays, 14:00-15:00 EDT (after weekly routines)
**Participants:** Lui (facilitator), Brains (co-facilitator), all Chiefs optional
**Duration:** 60 minutes maximum

## Process:
1. **Preparation (15 min before):**
   - Lui and Brains gather metrics and data
   - Pre-fill checklist with available information
   - Identify obvious issues for discussion

2. **Checklist Review (30 min):**
   - Systematically review each checklist item
   - Note issues, questions, and action items
   - Assign owners and deadlines for actions

3. **Action Planning (15 min):**
   - Prioritize identified issues
   - Create specific action plans
   - Update PROJECT_BOARD.md with health-related tasks

4. **Reporting (After meeting):**
   - Update SYSTEM_HEALTH.md with results
   - Share summary with all Chiefs
   - Escalate critical issues to User if needed

## Health Scoring System:
**Green (Healthy):** 0-1 issues per category
**Yellow (Watch):** 2-3 issues per category  
**Red (Action Required):** 4+ issues per category

**Overall Health Score:**
- **90-100%:** Excellent - system functioning optimally
- **75-89%:** Good - minor issues being addressed
- **60-74%:** Fair - needs attention, not critical
- **Below 60%:** Poor - immediate action required

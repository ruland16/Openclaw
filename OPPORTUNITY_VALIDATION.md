# OPPORTUNITY VALIDATION FRAMEWORK

Used by Warren (Strategy Chief).

========================
VALIDATION STEPS
========================

## 1. Define the Idea
**Purpose:** Create clear, concise definition of the opportunity
**Process:**
- Write one-sentence description: "We help [target user] achieve [benefit] by [solution]"
- Define core value proposition: What unique value does this provide?
- Identify key differentiators: Why is this better than alternatives?
- Estimate total addressable market (TAM): Rough size in dollars/users
**Output:** Opportunity Definition Document (1 page max)
**Quality Check:** Could you explain this to a stranger in 30 seconds?

## 2. Identify Target User
**Purpose:** Understand who will use/pay for this solution
**Process:**
- Create user persona(s): Demographics, behaviors, pain points
- Map user journey: How they discover, evaluate, purchase, use
- Identify acquisition channels: Where to find these users
- Estimate customer lifetime value (LTV): Revenue per user over time
- Determine willingness to pay: What price point makes sense?
**Output:** Target User Profile with acquisition and monetization insights
**Quality Check:** Could you point to 10 real people who fit this profile?

## 3. Define Monetization Model
**Purpose:** Determine how this opportunity generates revenue
**Process:**
- Select revenue model: SaaS subscription, one-time purchase, marketplace, ads, etc.
- Price point analysis: Competitor pricing, value-based pricing, cost-plus
- Revenue projection: Conservative, realistic, optimistic scenarios
- Cost structure: Fixed costs, variable costs, customer acquisition cost (CAC)
- Unit economics: LTV:CAC ratio (target >3:1), payback period
**Output:** Monetization Model with financial projections
**Quality Check:** Does the math work at scale? (LTV > 3x CAC)

## 4. Analyze Competition
**Purpose:** Understand competitive landscape and positioning
**Process:**
- Direct competitors: Solving same problem for same users
- Indirect competitors: Solving similar problems or different approaches
- Substitute solutions: How users solve this problem today
- Competitive analysis matrix: Features, pricing, strengths, weaknesses
- Market gaps: What competitors are missing or doing poorly
**Output:** Competitive Landscape Analysis with positioning strategy
**Quality Check:** What's your unfair advantage against each competitor?

## 5. Define "Why Now?"
**Purpose:** Identify timing factors that make this opportunity viable now
**Process:**
- Market trends: Growing demand, regulatory changes, technology shifts
- Technology readiness: Required tools/tech now available/affordable
- Consumer readiness: User behavior changes, adoption curves
- Competitive window: How long before competitors catch up?
- External factors: Economic conditions, funding environment, etc.
**Output:** Timing Analysis explaining why this is the right moment
**Quality Check:** Would this have worked 2 years ago? Will it work 2 years from now?

## 6. Create a 7‑Day Test Plan
**Purpose:** Design minimal experiment to validate key assumptions
**Process:**
- Identify riskiest assumption: What could kill this idea if wrong?
- Design MVP (Minimum Viable Product): Smallest thing to test assumption
- Define success metrics: Quantitative targets for validation
- Build measurement system: How to track results accurately
- Create execution timeline: Day-by-day plan for 7-day test
**Output:** 7-Day Validation Plan with clear metrics and execution steps
**Quality Check:** Can this test be completed in 7 days with <$500 budget?

## 7. Measure Results
**Purpose:** Collect and analyze data from validation test
**Process:**
- Daily tracking: Monitor key metrics throughout test
- User feedback: Qualitative insights from test participants
- Quantitative analysis: Did metrics hit success targets?
- Assumption validation: Which assumptions proved true/false?
- Learnings documentation: What surprised us? What worked/didn't?
**Output:** Validation Test Results Report with data and insights
**Quality Check:** Are results statistically significant? Are there confounding factors?

## 8. Decide: Scale, Pivot, or Kill
**Purpose:** Make data-driven decision about opportunity future
**Process:**

### Scale Decision (If validation successful):
- **Criteria:** Met or exceeded all success metrics, strong user feedback, positive unit economics
- **Actions:** 
  - Create detailed business plan
  - Allocate resources for full build
  - Add to PROJECT_BOARD.md as active project
  - Assign ownership to appropriate Chief(s)
- **Next Steps:** Move to execution phase with defined milestones

### Pivot Decision (If partial validation):
- **Criteria:** Some assumptions validated, others disproven, but core insight valuable
- **Actions:**
  - Identify which elements worked
  - Redefine opportunity based on learnings
  - Return to Step 1 with refined idea
  - Design new 7-day test for pivot
- **Next Steps:** Iterate through framework again with revised approach

### Kill Decision (If validation failed):
- **Criteria:** Key assumptions disproven, poor metrics, negative feedback, bad economics
- **Actions:**
  - Document learnings for future reference
  - Archive opportunity in legacy_archive.md
  - Reallocate resources to other opportunities
  - Share insights with other Chiefs to prevent similar mistakes
- **Next Steps:** Move to next opportunity in pipeline

**Output:** Strategic Decision with rationale and next steps
**Quality Check:** Is this decision based on data, not emotion or sunk cost fallacy?

========================
EXAMPLE VALIDATION
========================

### Opportunity: "Dishwasher Management App"
**Step 1 - Idea:** "We help restaurant owners reduce repair costs by predicting dishwasher failures before they happen using IoT sensors and AI."
**Step 2 - Target User:** Restaurant owners with 3+ locations, spending >$5k/year on dishwasher repairs
**Step 3 - Monetization:** $99/month SaaS subscription per location, LTV $2,376, CAC $300
**Step 4 - Competition:** Manual maintenance services, reactive repair companies, no predictive solutions
**Step 5 - Why Now:** IoT sensor costs dropped 80% in 3 years, AI prediction accuracy now >90%, restaurant tech adoption accelerating
**Step 6 - 7-Day Test:** Install sensors on 3 restaurant dishwashers, collect data, provide 1 failure prediction
**Step 7 - Results:** Predicted 2/3 failures correctly, restaurants willing to pay $79-$129/month
**Step 8 - Decision:** SCALE - Add to PROJECT_BOARD.md as IT project, assign to Elon for development
